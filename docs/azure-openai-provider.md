# Azure OpenAI Provider

The `ArchitectureGovernance.AI.AzureOpenAI` provider enables the generation of realistic, context-aware architecture artifacts using Microsoft's Azure OpenAI service.

## Configuration

To switch from the deterministic mock provider to the real Azure OpenAI provider, update the local configuration.

**`appsettings.Development.json` or User Secrets (`dotnet user-secrets`)**:

```json
{
  "AiProvider": {
    "Provider": "AzureOpenAI"
  },
  "AzureOpenAI": {
    "Endpoint": "https://your-resource.openai.azure.com/",
    "ApiKey": "YOUR_API_KEY",
    "DeploymentName": "gpt-4o",
    "TimeoutSeconds": 30
  }
}
```

> [!CAUTION]
> Never commit API keys or connection strings to source control. Always use environment variables, User Secrets, or Azure Key Vault for production secrets.

## Responsible AI Guidelines

All content generated by Azure OpenAI is strictly treated as draft. The system explicitly prepends a Human Review Notice to all output:

> "This artifact is AI-assisted draft content and must be reviewed by a qualified architect before use in production decisions."

## Cost Awareness

When deploying to Azure, be aware of token accumulation.
- Use `gpt-4o-mini` or smaller models for non-production environments to save costs.
- Set budget alerts within the Azure Subscription.
- Ensure the `Mock` provider is primarily used for local development and CI testing to prevent accidental burn rates.

## Safe AI Telemetry

The platform adheres strictly to safe telemetry rules in Azure Application Insights:
- Application Insights logs operation metadata (Duration, Template Name, User IDs).
- **No full prompt text** is ever logged to telemetry.
- **No full generated text** is ever logged to telemetry.
- **No requirement submissions** are written to Application Insights.

## Fallback & Resiliency

- If `AiProvider:Provider` is not set to `AzureOpenAI`, the system defaults to the `Mock` provider securely.
- If the Azure OpenAI configuration is missing or incomplete, the provider fails gracefully with a logged warning and a safe placeholder error string in the Markdown artifact output to avoid application crashes.
- A configurable timeout (`TimeoutSeconds`, defaults to 30s) prevents the backend from hanging on slow completions.
