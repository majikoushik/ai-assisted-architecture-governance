Epic 1 Goal:

Allow users to create, view, and manage architecture project workspaces.

An Architecture Project represents one solution, initiative, modernization program, or architecture review workspace. Future epics will attach requirement submissions, generated artifacts, prompt outputs, reviews, and AI interaction metadata to this project.

3. Add request and response DTOs.
4. Add validation for all required fields.
5. Add persistence using Entity Framework Core.
6. Add database migration support.
7. Add synthetic seed data if appropriate.
8. Add Swagger/OpenAPI documentation readiness.
9. Add structured logging.
10. Use correlation ID support if already available from Epic 0.
11. Add proper error handling using Problem Details or the existing error-handling pattern.
12. Add health check readiness if not already present.
13. Add unit tests and API/integration tests where practical.

Architecture Project fields:

* Project ID
* Project name
* Business domain
* Description
* Owner
* Status
* Created timestamp
* Updated timestamp

Supported project statuses:

* Draft
* Active
* UnderReview
* Archived

Validation rules:

* Project name is required.
* Project name should have a reasonable length limit.
* Business domain is required.
* Description is required.
* Owner is required.
* Status must be a supported value.
* Created timestamp should be system generated.
* Updated timestamp should be system generated or updated automatically.

* `POST /api/v1/projects`

* `PATCH /api/v1/projects/{projectId}/status`
