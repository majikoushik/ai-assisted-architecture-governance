Epic 4 Goal:

Allow the system to generate a Requirement Analysis artifact from a submitted requirement using the existing prompt catalog and deterministic mock AI provider.

This is the first end-to-end AI-assisted artifact generation workflow.

The generated output must be treated as draft architecture content requiring human architect review.

Suggested API endpoints:

* `POST /api/v1/artifacts/generate`
* `GET /api/v1/artifacts/{artifactId}`
* `GET /api/v1/projects/{projectId}/artifacts`
* `GET /api/v1/requirements/{requirementId}/artifacts`
* `GET /api/v1/artifacts/{artifactId}/export/markdown`

For this epic, `POST /api/v1/artifacts/generate` should support only:

* `RequirementAnalysis`

Other artifact types may return a clear validation or not-supported response until future epics.

Suggested generation request:

```json
{
  "projectId": "project-id",
  "requirementSubmissionId": "requirement-id",
  "artifactType": "RequirementAnalysis"
}
```

Use consistent API response and error response patterns according to `AGENTS.md`.
