Azure OpenAI configuration expectations:

Use environment-driven configuration such as:

```text
AI_PROVIDER=Mock
AZURE_OPENAI_ENDPOINT=<your-azure-openai-endpoint>
AZURE_OPENAI_API_KEY=<from-key-vault-or-user-secrets>
AZURE_OPENAI_DEPLOYMENT_NAME=<your-deployment-name>
AZURE_OPENAI_API_VERSION=<api-version>
AZURE_OPENAI_TIMEOUT_SECONDS=60
```

For local development:

```text
AI_PROVIDER=Mock
```

For future Azure deployment:

```text
AI_PROVIDER=AzureOpenAI
```

Use safe placeholder values only.

Do not commit real values.

Provider selection expectations:

1. `Mock` should be the default provider.
2. `AzureOpenAI` should be enabled only when explicitly configured.
3. If Azure OpenAI is selected but required configuration is missing, return a clear safe configuration error.
4. The application should not crash unexpectedly due to missing Azure OpenAI configuration.
5. The error response should not expose secrets or internal stack traces.
