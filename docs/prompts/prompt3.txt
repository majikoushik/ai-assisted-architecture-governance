Epic 2 Goal:

Allow users to submit business or technical requirements under an Architecture Project.

A Requirement Submission represents the raw business need, technical initiative, modernization request, or solution idea that will later be analyzed by the AI-assisted architecture engine.

Validation rules:

* Project ID is required.
* Project ID must refer to an existing Architecture Project if project lookup is available.
* Requirement title is required.
* Requirement title should have a reasonable length limit.
* Requirement text is required.
* Requirement text should have a minimum useful length.
* Requirement text should have a maximum length limit to avoid unsafe or excessive payloads.
* Business domain is required.
* Submitted by is required.
* At least one expected artifact type is required.
* Expected artifact types must be supported values.
* Status must be a supported value.
* Created timestamp should be system generated.
* Updated timestamp should be system generated or updated automatically.
