Epic 3 Goal:

Create a versioned prompt catalog and a deterministic mock AI provider so the application can generate realistic architecture artifact drafts locally without requiring Azure OpenAI credentials.

This epic prepares the foundation for future artifact generation epics.

The system must not require real Azure OpenAI configuration for local development.

* Return provider name

Mock AI provider expectations:

1. The mock provider must return deterministic, realistic, architecture-quality Markdown output.
2. Output should vary by artifact type.
3. Output should include assumptions, risks, open questions, and human review notice.
4. Output should not call any external API.
5. Output should be suitable for demos, tests, and local development.
6. Mock output should not pretend to be a real model response.
7. Mock output should clearly identify itself as generated by the mock provider.

Suggested prompt catalog API endpoints:

* `GET /api/v1/prompts`
* `GET /api/v1/prompts/{promptTemplateId}`

Suggested mock AI test endpoint, only if appropriate:

* `POST /api/v1/ai/mock/generate-preview`

Only add the mock preview endpoint if it fits the existing architecture cleanly. Otherwise, keep mock provider usage internal and document how it will be used in Epic 4.

Use consistent API response and error response patterns according to `AGENTS.md`.
